## Configuration

All configuration is via environment variables:

| Variable | Default | Description |
|---|---|---|
| `SERVICE_NAME` | `my-service` | Service identifier in logs and traces |
| `ENVIRONMENT` | `development` | Runtime environment |
| `PORT` | `8080` | HTTP listen port |
| `LOG_LEVEL` | `info` | Log level: debug, info, warn, error |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | `localhost:4317` | OTel collector endpoint |
| `CORS_ALLOWED_ORIGINS` | _(deny all)_ | Comma-separated allowed origins |
| `SHUTDOWN_TIMEOUT_SECONDS` | `30` | Graceful shutdown drain timeout |

## Tier 1 Standards Applied

- Structured JSON logging (W3C traceparent correlation)
- OpenTelemetry tracing (OTLP/gRPC export)
- RFC 7807 error responses
- Graceful shutdown (SIGTERM/SIGINT)
- Security headers (CSP, HSTS, X-Frame-Options)
- Input sanitization (path traversal, null bytes, body size)
- Non-root Dockerfile (distroless)
- Read-only root filesystem (K8s securityContext)
- Default-deny network policies
